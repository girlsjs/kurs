Arrays and loops

Loops

Today we're going to get familiar with loops.

Like conditional "if" statements, loops are one of the basic features of all programming languages.

They let us execute the same instructions for a few times, without having to copy-paste the same code.

Example?

Imagine we have 200 pages to photocopy on an oldschool machine that can handle only one page at a time.

Example

We're looking to delegate this task to an intern, as there's more interesting stuff for us to do.

Assuming that the intern has never seen a photocopier before and will only follow instructions w give them in print, how can we explain what's there for them to do?

Let's write these steps together

Most probably we've come up with something like


1. Open the photocopier lid.
2. Take the first sheet of paper from the stack.
3. Insert it into the photocopier, printed side down.
4. Close the lid.
5. Press the copy button.
6. Open the lid.
7. Take the sheet out of the photocopier and put it aside on a second stack (printed side down).
8. Take the second sheet of paper.
9. Insert it into the photocopier, printed side down.
10. (...)

We're not going to repeat these instructions for all 200 pages, now are we?

Let's try to optimize this!

We can see many instructions are repeating. We can bundle them together in a block and include instructions how many times this block needs to be repeated.

How do we write this dowm for our intern?


1. Open the photocopier lid.
2. Take the first sheet of paper.
3. Insert it into the photocopier, printed side down.
4. Close the lid.
5. Press the copy button.
6. Repeat the following for all pages 2 to 200:
  6.1. Open the lid.
  6.2. Take the sheet out of the photocopier and put it aside on a second stack (printed side down).
  6.3. Take the next sheet of paper from the stack.
  6.4. Insert it into the photocopier, printed side down.
  6.5. Close the lid.
  6.6. Press the copy button.
7. (Once we are done with page 200) Open the lid
8. Take the sheet out of the photocopier and put it aside on a second stack (printed side down).
9. Take the file of copied documents out of the photocopier drawer.
10. Bring both files to my office.

Step 6. in our algorithm is a loop

It consists of four elements:

  • Start conditions - In this example, our loop starts at the second page.
  • Stop conditions - Here we stop copying after page 200.
  • Looped instructions - everything covered by steps 6.x.
  • Transition to next step - Here we move from one page to another by increasing the number by one. This is not mandatory - if we copied only the even pages, we'd increase the number by 2.

First we'll want to learn a few facts about arrays.

Array

Array is a variable that can hold multiple values at once.

Variable

As we remember from our workshop, variable is a kind of storage, where we can save a value and then retrieve it later.

Array

If variable is a storage box, array is a whole wall of storage boxes, each with its own number assigned.

Why would we need that?

To store multiple values of the same kind.

Example: if we have 10 users on site, it's easier to create one array to store all of their names, than to store them in 10 different places.

How does it look like?

First a quick reminder how to create a variable:

var username = "Kasia";

And this is how we create an array of variables:

var usernames = ["Kasia", "Basia", "Asia"];

Or like this:

var usernames = [
  "Kasia",
  "Basia",
  "Asia"
];

How do we use it?

To read a regular variable, it's enough to use variable name. For example, this is how we'll send greetings to the console for the person whose name is stored in `username`:

console.log( "Hey " + username + "!" );

Similarly, to read the value of an array element, we'll need the array name and element number:

console.log( "Hey " + usernames[1] + "!" );

Watch out! Weird numbering ahead ;)

Array element numbers normally start from 0:

var names = [
  "Kasia", // This is number 0 element
  "Basia", // This is number 1 element
  "Asia"   // This is number 2 element
];

Loops

There are few different kinds of loops
Today we'll focus on one kind - the for loop.

It's the most widely used one.

The for loop

Remember our photocopier example?

for (var i=0; i<10; i++) {...}
  • the for keyword
  • start condition - creating a new variable
    (the loop needs information about itself)
  • stop condition
    (if the condition is no longer valid, the loop will stop)
  • transition to next step
    (what changes between the steps)
  • Looped instructions
    (in the brackets)

Le'ts compare this with conditional statement if

if

if (condition) {
  // do this if the condition is valid
  }

for

for (var i=0; i<10; i++) {
  // repeat this as long as the stop condition is not met
}

Let's type this code in the console

for (var i=0; i<5; i++) { console.log(i) }

Exercise

Exercise

Let's combine all we have learned together:

  • Create an array that contains 6 names.
  • Print all the names to the console within a loop

How does it look like?

var names = ["Ala", "Ela", "Ola", "Ula", "Ila", "Yla"];

for (var i=0; i<6; i++) {
  console.log(names[i]);
}

This is far from perfect, cause...

Our solution only works for 6 items in an array.

What if there's more or less of them?

"length" to the rescue!

Every array has a property called length

length always contains the current number of elements in an array.

names.length // prints out the number of elements

Back to our exercise

Let's modify the code we've written before in such a way so that it works with any number of names

Everything good so far?

var names = ["Ala", "Ela", "Ola", "Ula", "Ila", "Yla"];

for (var i=0; i<names.length; i++) {
  console.log(names[i]);
}

Awesome!
